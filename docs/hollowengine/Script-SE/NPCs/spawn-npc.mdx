# Creation and NPC's Spawn

## Template

:::info This is a template

```kts
val <npcID> by NPCEntity.creating{
    settings = NPCSettings("<displayName>", "hollowengine:<path/to/model>.gltf",   Attributes(<attributes>), Pair(<float1>, <float2>), <boolean>)
=======
val <npcID> by NPCEntity.creating {
    settings = NPCSettings(<nick>, <model>, <attributes>, <hitboxSize>, <showNick>)
    location = SpawnLocation("<dimension>", pos(<x>, <y>, <z>), vec(<pitch>, <yaw>))
}
```

- Designations:
> - `npcID` - Unique ID for your NPC. You use this ID to refer to a specific NPC.
> - displayName` - The name displayed above the NPC's head, similar to a player's nickname.
> - `path/to/model` - Specify the path to the model.
> - `attributes` - NPC attributes, such as health, speed, etc.
> - `Pair(<float1>, <float2>)` - NPC hitbox size.
>   - `float1` - Width.
>   - `float2` - Height.
> - `<boolean>` - Display of `displayName` above the NPC's head.
>   - `false` - Do not display.
>   - `true` - Display.
> - `dimension` - `dimension` - Dimension in which the NPC will spawn. Specify it with the `modID` first, for example: `minecraft:overworld`.
> - `pos(x, y, z)` - NPC spawn position.
> - `vec(pitch, yaw)` - Rotation angles when the NPC spawns.

- Optional Values:
> - `attributes`. 
> - `Pair(),` 
> - `showName`.


:::tip Hooray! Less writing!!
Instead of `Pair`, you can use the infix `to` operator, for example: `Attributes(<value1> to <value2>)`.
:::

## Basic NPC

:::info To spawn a basic NPC, leave the field `NPCSettings()` - empty.
Here is an example, how the "Basic NPC" spawn script should look like:
```kts
val npc by NPCEntity.creating{
    settings()
    location = SpawnLocation("minecraft:overworld", pos(0, -58, 0))
}
```

:::

## Custom NPC

:::info It is essential, that you specify the path to the model.
```kts
val npc by NPCEntity.creating{
    settings("Vitalik", "hollowengine:models/entity/temp2.gltf")
    location = SpawnLocation("minecraft:overworld", pos(0, -58, 0))
}
```
![Custom NPC's Spawn](../../.resourses/spawn-custom-npc.gif)
:::

### Attributes

:::info Every existing attributes, you can find using the command `/attribute`
In your case, you only need to specify where `attributes`, is, which is:
```kts
Attributes("<attributeName>" to <float>)
```
> Designations:
> - `attributeName` - name of the attribute.
> - `float` - Number/Digit, where in the end there is `f`, example: `10f`
:::
