## Импорт других скриптов

:::info Импорт других скриптов
> Если ваш сюжет состоит из нескольких частей, то может потребоваться использовать переменную из других частей.  
> Вы можете импортировать другие скрипты, и использовать из ник к примеру функции.
```jsx
@file:Import("<path/to/script>")
```
> Есль небольшое но:
> - Вы можете импортировать скрипт только из этого места, к примеру: `my_script_folder/functions/fun_pack.kts`. Вы не можете вернуться обратно чтобы взять скрипт из прошлой папки `../other_script_folder/test.kts`.
> - Т.е. "Импортируемый скрипт должен быть ниже той папки, где вы импортируете!"
>  - К примеру: У меня есть разделы `hollowengine/scripts/my_folder/` и в нём есть скрипт `script.se.kts`.  
>  - Мне нужно в этом скрипте импортировать другой скрипт, который находится в пути `hollowengine/scripts/my_folder/functions/text.kts`.  
>  - И чтобы мне импортироватьего, я пропишу в скрипте `script.se.kts` это:
```jsx
@file:Import("finctions/test.kts")
// И что же я сделал?
// - Я импортирую скрипт с помощью функции @file:Import(}
// - В скобках () - я указывают String значение.
// - В ковычках "" - я указываю путь, начитая с этой папки, но не указываю её.
```
:::

## Использование классических команд

:::info
> Бывает нужно прописать какую-нибудь классическую команду (как в командных блоках).  
> > - Необязательно ставить в мире командный блок и выщитывать время.  
> Можно просто использовать команду:
```jsx
execute{"<command"}
// Она работает как команды в командных блоках
// Не обязательно писать через / 
```
---
> Если в командах вам нужно использовать НИПа, используй это:
```jsx
npcID().uuid
// Как это применить?

execute{"effect give $npcID().uuid levitation 10"}
// - Выдаём эффект левитации по UUID-сущности
// - НИП это тоже сущность со своим UUID
// - через метод npcID().uuid мы получает UUID сущности
// - используя знак $ перед методом, мы используем переменную
```
:::
